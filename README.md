## Анализ данных интернет‑магазина «Подарочек»

Проект выполнен в рамках тестового задания Яндекс Практикума. Цель — оценить деятельность интернет‑магазина подарков на основе транзакционных данных и сформировать рекомендации по улучшению работы с клиентами.

### Данные

Используются два открытых датасета:

- **Транзакции (`gift.csv`)**  
  - `entry_date` — дата и время операции  
  - `order_id` — идентификатор заказа  
  - `customer_id` — идентификатор клиента (в т.ч. технические id для анонимных продаж)  
  - `quantity` — количество единиц товара  
  - `price` — цена за единицу  
  - `name_clust` — автоматически присвоенный кластер по наименованию товара  
  - `entry_id` — идентификатор товарной позиции  
  - `country_id` — идентификатор страны  

- **Справочник товаров (`gift_entry.csv`)**  
  - `entry_id` — идентификатор товарной позиции  
  - `entry` — текстовое наименование товара  

Данные приводятся к единым типам, справочник мэппится к транзакциям по `entry_id`, технические аномалии (`entry_id = 0` и др.) обрабатываются отдельно.

### Задачи анализа

**1. Загрузка и объединение данных**

- Импорт csv‑файлов в `pandas`  
- Анализ типов, пропусков и дубликатов  
- Нормализация строковых полей, удаление явных и неявных дублей  
- Проверка согласованности идентификаторов и объединение транзакций со справочником товаров

**2. Предобработка и исследовательский анализ**

- Поиск выбросов и аномалий в `quantity` и `price`:
  - анализ описательных статистик и boxplot‑диаграмм
  - оценка верхних квантилей (`0.99`, `0.995`, `0.999`)
  - выделение технических операций (списания, корректировки, комиссии)
- Разделение выборки:
  - полный датасет для расчёта выручки
  - подмножество с валидными клиентскими `customer_id` для RFM и ретеншн‑аналитики
- Анализ полноты данных по времени:
  - дни без продаж по месяцам
  - выбор репрезентативного периода для детального анализа

**3. Расчёт ключевых метрик**

- Почасовая и понедельная активность:
  - количество заказов
  - число уникальных клиентов
  - визуализация сезонных и недельных паттернов (heatmap, линии)
- Помесячные показатели:
  - средняя выручка с клиента в день
  - число уникальных покупателей
  - оценка сезонности и трендов
- Стики‑фактор (retention) за II и III кварталы 2019 года

**4. Построение клиентских профилей**

Для каждого клиента рассчитываются:

- количество заказов  
- дата первой и последней покупки  
- суммарная выручка  
- средний чек и средняя выручка на заказ  
- дополнительные поведенческие показатели (частота, давность и др.)

На основе профилей клиенты разделяются на возвратных и невозвратных, для каждой группы оцениваются средние метрики и вклад в выручку.

**5. RFM‑сегментация**

- Расчёт Recency, Frequency, Monetary для идентифицированных клиентов  
- Присвоение RFM‑оценок и формирование сегментов (например, 111, 333, 223 и др.)  
- Оценка поведения ключевых групп (частота покупок, средний чек, жизненная ценность)  
- Формирование практических рекомендаций по работе с сегментами:
  - удержание высокоценных клиентов (программы лояльности, персональные предложения)
  - реактивация «уснувших»
  - стимулирование роста среднего чека и частоты в перспективных сегментах

**6. Проверка статистических гипотез**

Используются непараметрические и параметрические тесты из `scipy` и `statsmodels`:

- **Сравнение долей возвратных клиентов**  
  - z‑тест для двух независимых пропорций (II vs III квартал 2019 года)  
  - формулировка односторонней гипотезы, интерпретация результата

- **Сравнение средних чеков по странам**  
  - выбор стран по `country_id` (3, 6, 24)  
  - проверка предпосылок для параметрических тестов  
  - непараметрический тест Краскела–Уоллиса  

- **Дополнительная гипотеза**  
  - формулировка бизнес‑гипотезы на основе EDA  
  - проверка с использованием подходящего статистического критерия

**7. Дополнительное исследование**

- Формирование «корзин» по заказам  
- Подсчёт пар товаров, наиболее часто встречающихся вместе  
- Интерпретация результатов в контексте кросс‑продаж и комплементарных предложений

### Основные выводы (кратко)

В проекте сформулирован развёрнутый итоговый вывод, включая:

- поведение спроса по времени суток и дням недели;
- наличие/отсутствие выраженной сезонности;
- структуру клиентской базы (доля возвратных, ключевые сегменты по RFM);
- статистически подтверждённые различия в поведении клиентов по кварталам и странам;
- приоритетные сегменты и направления для роста выручки (удержание, допродажи, реактивация).

### Технологический стек

- **Язык и окружение**: Python, Jupyter Notebook  
- **Обработка данных**: `pandas`, `numpy`  
- **Визуализация**: `matplotlib`, `seaborn`, `plotly`  
- **Статистический анализ**:  
  - `scipy.stats` (Kruskal–Wallis, Spearman и др.)  
  - `statsmodels.stats.proportion` (z‑тест пропорций)  
- **Счетчик обработки парных значений**: `collections`, `itertools`
